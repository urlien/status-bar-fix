## 状态栏崩溃问题排查

### 问题描述
用户在一个角色互动页面（斯卡蒂/Arknights相关）中，有三个按钮（摸头发、擦头发、询问），点击后通过 JS 往 textarea 填入预设文本。**添加引号后状态栏崩溃。**

### 根因
第三个按钮"询问"的 `t.value='...'` 字符串中，**开头是弯引号 `'`，结尾是直引号 `'`**，引号不匹配导致 JS 语法解析失败，页面/状态栏崩溃。

### 三个按钮代码对比

| 按钮 | 开头引号 | 结尾引号 | 状态 |
|------|---------|---------|------|
| 摸头发 | `'` (弯) | `'` (弯) | ✅ 正常 |
| 擦头发 | `'` (弯) | `'` (弯) | ✅ 正常 |
| 询问 | `'` (弯) | `'` (**直**) | ❌ 崩溃 |

### 修复方法
把第三个按钮结尾的直引号 `'` 改成弯引号 `'`：

```javascript
// 修复前（崩溃）
...现在我只想待在这里。"';

// 修复后（正常）
...现在我只想待在这里。"';
```

### 页面结构
- 三个按钮 CSS 类名: `skd-opt-b`
- 交互方式: `onclick` + `event.stopPropagation()`
- 目标: `document.querySelector('textarea')`
- 触发事件: `new Event('input', {bubbles: true})` 通知框架内容已更新
