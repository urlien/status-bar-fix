# 2026-06-11

## 状态栏引号崩溃 - 完整排查与修复记录

### 背景
延续 2026-06-10 的排查。之前只修了第三个按钮的 U+2018/U+0027 不匹配问题，用户反馈**只要选项内容里有引号就会崩溃**。

### 根因分析

`替换内容.txt`（HTML 模板）中三个按钮的 JS 代码用单引号包裹变量：

```html
<div class="skd-opt-b" onclick="...t.value='$20';...">$19</div>
```

`$20` 被正则替换为选项内容后，内容里的任何引号都会导致 JS 字符串提前闭合，SyntaxError，状态栏崩溃。

### 尝试过的方案

| 方案 | 结果 | 原因 |
|------|------|------|
| 修单个引号不匹配 (U+2018→U+2019) | ❌ 不够 | 只修了一种情况 |
| 单引号改反引号 `t.value=`...`` | ❌ 不够 | AI 文本可能含反引号或 `${` |
| 用 『 』 替代引号 | ❌ 不够 | `『` `』` 含特殊字符，正则替换后仍可能破坏 HTML |
| **AI prompt 禁止所有引号+特殊字符** | ✅ 可行 | 从源头控制，文本干净就不会崩 |

**最终结论：从 prompt 层面控制是唯一可靠方案。** 状态栏文本中不能出现任何引号和特殊字符。

### 指令集说明

`斯卡蒂状态栏_平台AI指令集.txt` 是最终使用的方案，直接作为 platform prompt 喂给 AI：
- 定义状态栏完整格式（所有字段、顺序、取值说明）
- 所有身体状态用形容词，不用数字
- 选项内容必须纯叙述，严禁任何引号
- 内心独白一句话，不用引号包裹
- 所有字段必须填写不得留空
