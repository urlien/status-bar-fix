# 2026-06-11

## 状态栏引号崩溃 - 深度排查与修复

### 背景
延续 2026-06-10 的排查。之前只修了第三个按钮的 U+2018/U+0027 不匹配问题，但用户反馈**只要选项内容里有引号就会崩溃**，不只是那一个按钮。

### 根因分析

`替换内容.txt`（HTML 模板）中三个按钮的 JS 代码用**单引号 `'...'`** 包裹变量：

```html
<div class="skd-opt-b" onclick="...t.value='$20';...">$19</div>
<div class="skd-opt-b" onclick="...t.value='$22';...">$21</div>
<div class="skd-opt-b" onclick="...t.value='$24';...">$23</div>
```

`$20`、`$22`、`$24` 被正则替换为选项内容后，如果内容里包含任何引号（`'` `"` `'` `'` 等），JS 字符串就会提前闭合或语法错误，导致整个 `<script>` 块执行失败，状态栏崩溃。

**本质上不是某个引号字符的问题，而是整个方案本身就脆弱。**

### 修复方案

#### 方案一：替换内容用反引号（JS 层面）
把 `t.value='$20'` 改成 `` t.value=`$20` ``（模板字面量），这样单引号和双引号都不会冲突。

```diff
- t.value='$20'
+ t.value=`$20`
```

**文件：** `替换内容_fixed.txt`

#### 方案二：AI 输出规则禁止引号（prompt 层面）
从源头控制，让 AI 在输出状态栏文本时把所有引号替换为 『 』。

```
禁止使用的字符：'  '  '  "  "  "  `
统一用 『 』 替代所有引号
```

**文件：** `状态栏规则_禁止引号.md`

#### 最终建议
两个方案都用：
1. `替换内容_fixed.txt` 部署到实际使用的地方（防御层）
2. `状态栏规则_禁止引号.md` 加到 AI prompt 里（源头控制）

1. 用户提供三个文件：
   - `状态栏文本.txt` — 状态栏文本模板（含占位符）
   - `替换内容.txt` — HTML 模板（CSS + JS）

3. 生成 `替换内容_fixed.txt`（反引号版本）

5. 全部文件推送到 `urlien/status-bar-fix` 仓库
